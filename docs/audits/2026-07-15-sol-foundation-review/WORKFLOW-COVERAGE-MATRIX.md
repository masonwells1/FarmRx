## Cross-module observations

- **Harvest -> Grain:** The UI now shows both values and offers an explicit “Use harvest total as Grain actual” action. This is safer than silent mutation, but the Grain number remains stale until a farmer performs the reconciliation.
- **Programs -> Inventory:** Marking a free-type Program pass applied can create/link a draft application record without changing on-hand inventory. This is explicit and avoids pretending free text is a catalog match.
- **Inventory -> Profitability:** Saved application/product costs can feed reporting, but queued offline bundles are not fully projected into the Inventory workspace.
- **Equipment/Programs -> Tasks/Alerts:** Database authority prevents task-board edits from falsely completing Program passes; due generation and delivery timing still depend on an evaluator being invoked.
- **Fields/leases -> reports:** Same-farm foreign keys and structured arrangement history are strong; stale full-record saves remain the main reconciliation risk.
