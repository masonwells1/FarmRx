# Adversarial Review — Module 1 Fields Polish

1. **P1 (must fix) — Failed persistence is reported as “Saved.”** `src/data/MockFieldsRepository.ts:164` catches every `localStorage.setItem` failure and still lets `saveField()` resolve successfully. `src/FieldsModule.tsx:71` then clears the quick-add inputs and displays “Saved,” although reloading restores the old data. This occurs when storage is full, disabled, or throws. **Concrete fix:** make `persist()` throw on write failure and verify the serialized value was stored before `saveField()` resolves; only clear the row and announce success after that confirmed result.

2. **P1 (must fix) — Unsupported or corrupt envelopes can erase the Grain compartment.** `src/data/MockFieldsRepository.ts:135-160` ignores `grain` when parsing fails or the envelope version is not exactly 2, then `load()` replaces Fields with seed data and calls `persist()`, overwriting the shared key without Grain. That violates the hard rule that no Fields save path may touch Grain. **Concrete fix:** fail closed when the existing shared envelope cannot be safely parsed/preserved; never rewrite it from the Fields repository unless its non-Fields compartments have been copied byte-for-byte. Add corrupt-JSON and unsupported-version regression cases.

3. **P1 (must fix) — Existing history prevents adding another crop assignment.** `src/FieldsModule.tsx:107-108` renders the new-record controls only when `rows.length === 0`. A field with any historical assignment—but no current-year crop—can be identified by the KPI nudge but cannot be given the missing current-year assignment from the Records card. It also prevents adding a second crop or future season. **Concrete fix:** always provide an “Add crop record” row, append it to the existing records, validate unique year/commodity/sequence keys, and preserve all untouched history.

4. **P1 (must fix) — Equivalent-rent output can use an arbitrary price and misstate multi-crop rent.** `src/FieldsModule.tsx:38` picks the first cash bid after sorting only by date; two same-day bids produce an array-order-dependent price with no elevator or delivery scope. `src/FieldsModule.tsx:95-96` then emits one complete “equivalent cash rent” per crop rather than one field-level total; cash rent is duplicated for every assignment, while crop-share/flex double-crop economics are never aggregated. **Concrete fix:** require the manual planned price unless a canonical, explicitly sourced Grain price is available; display its source, calculate one field-level result with base rent applied once, and aggregate crop-dependent components deliberately.

5. **P2 (should fix) — Flex estimates require irrelevant inputs and hide the units of the terms.** `src/FieldsModule.tsx:40-49` rejects every flex estimate unless both yield and price exist, even though price-trigger terms need only price and yield-trigger terms need only yield. At `src/FieldsModule.tsx:96`, generic “Flex trigger” and “Flex bonus rate” labels do not tell users whether values mean `$ /bu`, `bu/ac`, `$ /ac`, or `%`. **Concrete fix:** validate inputs by formula type and dynamically label every trigger and rate with its actual unit.

6. **P2 (should fix) — The KPI filter can contradict its own count.** `src/FieldsModule.tsx:55-64` intersects the “missing crop” toggle with the existing crop selector. If a crop is selected first, clicking “Crops assigned x/y” necessarily shows zero rows because a field cannot both have that crop and be missing a crop. **Concrete fix:** clear `cropFilter` when enabling the nudge or make the missing-crop mode override the crop selector.

7. **P2 (should fix) — Concurrent card editing leaves Yield & price unable to save.** `src/FieldsModule.tsx:99-101` keys unsaved values by assignment ID. Saving Records at `src/FieldsModule.tsx:107` replaces assignments with new IDs; an already-open Yield editor then finds no matching values, converts `undefined` to `NaN`, and fails validation. **Concrete fix:** preserve assignment IDs on updates, or key editing state by the repository’s stable year/commodity/sequence identity and reconcile refreshed data without discarding user input.

8. **P2 (should fix) — The required regression is helper-only and not executable through the project scripts.** `src/data/MockFieldsRepository.regression.ts:5-22` directly exercises serialization helpers but never calls `MockFieldsRepository.saveField()`, never simulates a storage write failure, and is not invoked by any `package.json` script. It therefore cannot catch the false-“Saved” path, actual contact saves, close-and-insert regressions, or Grain loss on malformed envelopes. **Concrete fix:** add an executable regression runner that uses the repository with controlled storage and asserts contact round-trip, unchanged Grain, failure propagation, same-date update, and future-effective close-and-insert behavior.

9. **P2 (should fix) — The new “Needs crop” text violates the 18px design minimum.** `src/FieldsModule.tsx:67` introduces `.status-chip` in the Fields list, while `src/styles/app.css:248` sets `.status-chip` to exactly `14px`. **Concrete fix:** add a Fields-specific selector such as `.fields-page .status-chip { font-size: 18px; }`; retain at least 48px only if the chip becomes interactive.

10. **P2 (should fix) — Quick-add confirmation is not announced to assistive technology.** `src/FieldsModule.tsx:73` inserts “Saved” or an error as an ordinary `<span>` after focus has already left the row, with no `role="status"`, `role="alert"`, or live region. Keyboard and screen-reader users may never learn whether blur-save succeeded. **Concrete fix:** use a persistent `aria-live="polite"` status region and `role="alert"` for validation failures.

Strict TypeScript checking passed with `tsc -p tsconfig.app.json --noEmit`. A live visual pass could not be captured because no local preview was running and this review was constrained to read-only operation.

VERDICT: NEEDS FIXES (4 P1)
